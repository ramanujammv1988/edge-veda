  <name>Task 1: Create CI workflow documentation</name>
  <files>docs/CI.md</files>
  <action>
  Create `docs/CI.md` documenting the CI pipeline. Read the final state of `.github/workflows/ci.yml` (after Plan 01 changes) to ensure accuracy.

  Structure the document as follows:

  **## CI Pipeline Overview**
  One paragraph explaining the CI runs on push to main/develop and on PRs.

  **## Job Matrix**
  A table with columns: Job | Status | Blocks Merge | Runs On | Purpose
  - `core-build` | Required | Yes | ubuntu + macos, Debug + Release | Builds C++ core, runs ctest, checks binary size
  - `flutter-analyze` | Required | Yes | ubuntu | Format check, static analysis, unit tests, coverage upload
  - `code-quality` | Required | Yes | ubuntu | clang-format, large file check, license headers
  - `security-scan` | Required | Yes | ubuntu | Trivy filesystem scan
  - `ios-build` | Optional | No | macos | Builds iOS XCFramework (continue-on-error: true). Requires Xcode Metal and local submodules.
  - `android-build` | Optional | No | ubuntu | Builds Android AAR (continue-on-error: true). Requires NDK and local submodules.
  - `integration-tests` | Main only | No (informational) | macos | Runs Flutter integration smoke test. Only on main branch merges.
  - `build-summary` | Always | No | ubuntu | Aggregates results from all jobs.

  **## Why Some Jobs Are Optional**
  Explain: iOS and Android builds require platform SDKs (Xcode Metal toolchain, Android NDK) and git submodules that are large (llama.cpp, whisper.cpp, stable-diffusion.cpp). These builds validate that native compilation succeeds but cannot block unrelated Dart or C++ changes. The `build-summary` job reports their status for visibility.

  **## Required Checks for Branch Protection**
  List the jobs that should be configured as required status checks in GitHub branch protection:
  - `Core Build - ubuntu-latest / Release`
  - `Core Build - macos-latest / Release`
  - `Flutter Analyze & Test`
  - `Code Quality`
  - `Security Scan`
  - `Build Summary`

  Note: These are the `name:` values from ci.yml that GitHub uses for status checks. Include a note about matrix jobs generating multiple check names.

  **## Coverage**
  - Flutter coverage is uploaded to Codecov via `codecov/codecov-action@v4`
  - Coverage comments appear on PRs via `codecov.yml` configuration
  - Generated FFI bindings are excluded from coverage metrics
  - Flag: `flutter` tracks Flutter-specific coverage

  **## NDK Version**
  - Pinned to r27c (27.2.12479018) across ci.yml, build-android.sh, and build.gradle
  - r27c chosen because r28+ has 16KB page alignment issues affecting mmap-based model loading
  - All three files must stay in sync when upgrading

  **## Adding New CI Jobs**
  Brief guidance: if the job is deterministic (no hardware dependency), make it required. If it depends on platform SDKs or physical devices, make it optional with `continue-on-error: true`.
  </action>
