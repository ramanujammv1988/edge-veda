<task type="auto">
  <name>Task 2: Create Codecov configuration for PR coverage comments</name>
  <files>codecov.yml</files>
  <action>
  Create `codecov.yml` at the repo root with the following configuration:

  - Set `coverage.status.project.default` with `target: auto` and `threshold: 1%` (allows 1% regression before failing). This means Codecov will track coverage trends without being overly strict.
  - Set `coverage.status.patch.default` with `target: 80%` so new code in PRs should have reasonable coverage.
  - Enable `comment.layout: "reach,diff,flags,files"` to show coverage diff and file-level breakdown on PRs.
  - Set `comment.behavior: default` (posts new comment, updates existing).
  - Set `comment.require_changes: false` so comment always appears.
  - Add `flags.flutter` with `paths: ["flutter/lib/"]` and `carryforward: true` to track Flutter coverage separately.
  - Set `ignore` to exclude generated files: `["flutter/lib/src/ffi/generated_bindings.dart"]`.

  The existing ci.yml already uploads coverage via `codecov/codecov-action@v4` with `flags: flutter`, so the flag name `flutter` must match exactly.

  Note: The codecov-action in ci.yml does not currently specify a `token`. For public repos, Codecov works without a token. If the repo is private, the user will need to add a `CODECOV_TOKEN` secret. Do NOT add token configuration to the action — this is fine for a public repo.
  </action>
  <verify>
  Run: `cat codecov.yml` and confirm it has valid YAML with coverage.status, comment, flags, and ignore sections.

  </verify>
  <done>codecov.yml exists at repo root with project/patch thresholds, PR comment configuration, flutter flag matching ci.yml, and generated file exclusion.</done>
