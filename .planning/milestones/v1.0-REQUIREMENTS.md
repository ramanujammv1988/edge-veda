# Requirements Archive: v1.0 Flutter Android Support

**Archived:** 2026-02-28
**Milestone:** v1.0

## 1. Core Platform Support
- [x] **Must Have:** Expose Android plugin support for Edge Veda via `pubspec.yaml`.
  - **Outcome:** Validated — pubspec.yaml already declared Android platform; build.gradle configured with externalNativeBuild pointing to core/CMakeLists.txt
- [x] **Must Have:** Bind C++ Core Functions (`ev_*`) to Android native environments.
  - **Outcome:** Validated — 9 JNI functions in bridge_jni.cpp, 48/48 ev_* symbols bound in Dart FFI, DynamicLibrary.open('libedge_veda.so') for Android
- [x] **Must Have:** Graceful CPU-only fallback with OpenMP safely disabled due to broad NDK version instability parsing dynamic page limits.
  - **Outcome:** Validated — EV_BACKEND_CPU enforced in JNI bridge, GGML_OPENMP=OFF in CMake and Gradle, GGML_LLAMAFILE=OFF, Vulkan OFF

## 2. CI/CD & Compilation
- [x] **Must Have:** Provide `build-android.sh` to cross-compile the NDK toolchain cleanly using CMake.
  - **Outcome:** Validated — 243-line production script with --clean/--debug/--release/--help, symbol verification (ev_* >= 20, JNI >= 5, llama.cpp >= 10), submodule auto-init
- [x] **Must Have:** Automate `.so` dynamic library packaging in gradle setup constraints to avoid 16KB-page faults.
  - **Outcome:** Validated — 16KB page alignment verified via readelf, linker flags set in CMakeLists.txt, jniLibs.srcDirs = [] prevents conflicts

## 3. Platform APIs
- [x] **Must Have:** Wire Android specific `AudioRecord` APIs to pass raw PCM feeds to whisper stt implementations.
  - **Outcome:** Partially validated — 5 whisper JNI functions implemented (init, transcribe, free, version, isValid), Kotlin external declarations ready, runtime AudioRecord integration deferred to on-device testing
- [x] **Must Have:** Route `ActivityManager` telemetry up the Kotlin JNI bridge so the Dart layer calculates memory caps appropriately avoiding Android OS OOM termination thresholds.
  - **Outcome:** Validated — Telemetry MethodChannel with getAvailableMemory (ActivityManager.MemoryInfo.availMem) + getDeviceMemoryInfo (totalMem, availMem, lowMemory, threshold), getMemoryRSS (Debug.getNativeHeapAllocatedSize)
